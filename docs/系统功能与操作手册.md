# 高校信息系统账号管理平台

## 功能设计与使用操作手册

**版本**：1.0  
**适用对象**：系统管理员、院系管理员、科室负责人、普通教职工  
**访问地址**：http://127.0.0.1:5001（本地部署默认端口）

---

## 一、系统概述

### 1.1 建设目标

本平台面向**高校信息化场景**，用于集中管理校内各信息系统的登录账号与密码，例如：

- 教务管理系统、学工系统、财务系统
- OA 办公、图书馆、邮件、VPN
- 一卡通、站群、科研/人事系统等

系统支持**分级多用户**、**院系部门组织架构**、**密码加密存储**与**操作审计**，满足高校多部门协同与信息安全管理需求。

### 1.2 技术特性

| 项目 | 说明 |
|------|------|
| 开发语言 | Python 3.9+ |
| Web 框架 | Flask |
| 数据库 | SQLite（默认，可换 PostgreSQL/MySQL） |
| 密码存储 | 登录密码哈希；系统账号密码 Fernet 加密 |
| 登录安全 | 4 位数字验证码（Session 一次性校验） |

## 二、功能设计

### 2.1 功能模块总览

```
┌─────────────────────────────────────────────────────────────┐
│                    高校信息系统账号管理平台                    │
├──────────────┬──────────────┬──────────────┬──────────────────┤
│   用户认证    │  系统账号库   │   后台管理    │    工作台/首页    │
│  登录/登出    │  增删改查     │  用户/部门    │    统计概览      │
│  数字验证码   │  搜索筛选     │  角色/审计    │                  │
└──────────────┴──────────────┴──────────────┴──────────────────┘
```

### 2.2 系统账号库模块

**核心实体：系统账号记录**

| 字段 | 说明 | 必填 |
|------|------|------|
| 系统名称 | 如「教务管理系统」「校园 VPN」 | 是 |
| 账户 | 登录用户名或工号 | 是 |
| 密码 | 加密存储，列表页不显示明文 | 是（新建时） |
| 网址 | 系统访问 URL | 否 |
| 使用部门 | 如教务处、信息中心 | 是 |
| 系统类型 | 预置 14 类高校常见系统 | 否 |
| 备注 | 用途、变更说明、注意事项 | 否 |
| 更新时间 | 每次保存自动更新 | 自动 |
| 可见范围 | 私有 / 组织内 / 组织及下级 | 管理员可设 |

**子功能：**

- 列表展示、关键词搜索、按使用部门筛选
- 新增 / 编辑 / 删除 / 详情查看
- 密码显示（需点击「显示」）、复制
- 对指定用户单独授权（查看或编辑）

### 2.3 后台管理模块（`/admin`）

| 子模块 | 路径 | 功能 |
|--------|------|------|
| 管理概览 | `/admin/` | 用户/部门/账号统计、角色分布、最近操作 |
| 分级用户 | `/admin/users` | 新建、编辑、启用/禁用；按角色、部门筛选 |
| 院系部门 | `/admin/departments` | 组织树、部门明细；新建、编辑、删除 |
| 角色权限 | `/admin/roles` | 5 级角色说明 |
| 操作审计 | `/admin/audit-logs` | 登录、增删改等操作记录（分页） |

### 2.4 用户认证模块

- 用户名 + 密码登录
- **4 位数字验证码**（错误后自动刷新，支持手动刷新）
- 会话保持，退出后清除

### 2.5 组织架构设计

采用**树形多级结构**：

```
某某大学（顶级）
├── 信息中心
├── 教务处
├── 学生工作处
├── 财务处
├── 计算机学院
└── 图书馆
```

部门用于：用户归属、账号的使用部门、权限范围控制。

### 2.6 角色与权限设计

角色等级**数字越小，权限越高**：

| 等级 | 角色 | 主要权限 |
|:----:|------|----------|
| 0 | 系统管理员 | 全校用户、部门、全部系统账号；无限制 |
| 1 | 院系管理员 | 管理本院系及下级部门用户；可查看部门内共享账号 |
| 2 | 科室负责人 | **可新增**系统账号；**仅可见本人添加**的记录 |
| 3 | 普通教职工 | 查看授权范围内账号，不可新增 |
| 4 | 只读用户 | 仅查看，不可修改 |

**系统账号可见范围（管理员创建时可设置）：**

| 范围 | 说明 |
|------|------|
| 私有 | 仅创建者 + 被单独授权的用户 |
| 组织内可见 | 同部门内按角色可见 |
| 组织及下级可见 | 本部门及所有下级部门可见 |

**科室负责人特殊规则：**

- 新建记录自动设为「私有」，绑定本科室
- 不能查看部门内他人创建的共享记录
- 只能编辑、删除自己创建的记录

### 2.7 安全设计

- 系统账号密码：**Fernet 对称加密**后入库
- 用户登录密码：**PBKDF2-SHA256** 哈希
- 验证码：Session 存储，验证后立即销毁
- 操作审计：记录登录、查看、创建、修改、删除、授权等

## 四、使用操作手册

### 4.1 系统启动

```bash
cd ~/Documents/vault-platform
source .venv/bin/activate
python run.py
```

浏览器访问：**http://127.0.0.1:5001**

> macOS 默认占用 5000 端口（AirPlay），本系统使用 **5001**。

---

### 4.2 登录与退出

**登录步骤：**

1. 打开首页，点击「登录平台」
2. 输入用户名、密码
3. 输入右侧显示的 **4 位数字验证码**
4. 验证码看不清时，点击「刷新」
5. 点击「进入系统」

**退出：** 点击导航栏右上角「退出」

**常见提示：**

| 提示 | 原因 | 处理 |
|------|------|------|
| 验证码错误 | 输入不匹配或已过期 | 刷新验证码后重试 |
| 用户名或密码错误 | 账号或密码不正确 | 核对演示账号或联系管理员 |

---

### 4.3 工作台

登录后默认进入 **工作台**（`/dashboard`），可查看：

- 当前账号可访问的系统账号数量
- 平台用户总数、院系部门数
- 最近更新的系统账号
- 最近操作审计摘要

---

### 4.4 系统账号库操作

#### 4.4.1 查看列表

1. 点击导航栏 **「系统账号库」**
2. 使用搜索框：按系统名称、账户、网址、部门、备注搜索
3. 使用「全部使用部门」下拉框筛选

#### 4.4.2 新增记录（科室负责人及以上）

1. 点击 **「新增记录」**
2. 填写表单：

   | 字段 | 说明 |
   |------|------|
   | 系统名称 | 必填 |
   | 系统类型 | 下拉选择 |
   | 账户 | 必填 |
   | 密码 | 必填（新建时） |
   | 网址 | 可选 |
   | 使用部门 | 必填；科室负责人自动绑定本科室 |
   | 关联院系 | 管理员可设权限范围 |
   | 可见范围 | 管理员可选（科室负责人固定为私有） |
   | 备注 | 可选 |

3. 点击 **「保存」**

#### 4.4.3 查看详情

1. 在列表中点击 **「详情」**
2. 密码默认隐藏，点击 **「显示」** 查看明文
3. 点击 **「复制」** 可复制密码（需先显示）

#### 4.4.4 编辑 / 删除

1. 在详情页点击 **「编辑」** 或 **「删除」**
2. 编辑时密码留空表示不修改
3. 仅具备编辑权限的用户可见上述按钮

#### 4.4.5 授权其他用户（有编辑权限时）

1. 在详情页右侧「授权其他用户」区域
2. 选择用户，可选「允许编辑」
3. 点击 **「授权」**；已授权用户可 **「撤销」**

---

### 4.5 后台管理操作（院系管理员及以上）

入口：导航栏 **「后台管理」** → http://127.0.0.1:5001/admin

#### 4.5.1 管理概览

查看管辖范围内的用户、部门、账号统计及快捷入口。

#### 4.5.2 分级用户管理

1. 进入 **「分级用户」**
2. 可按用户名/邮箱、角色、部门筛选
3. **新建用户**：设置用户名、邮箱、密码、角色等级、所属部门
4. **编辑用户**：修改邮箱、角色、部门、密码；可启用/禁用账号
5. 规则：不能创建或编辑比自己权限更高的用户

#### 4.5.3 院系部门管理

1. 进入 **「院系部门」**
2. 左侧查看组织树，右侧查看部门明细（用户数、账号数）
3. **新建部门**：填写名称、选择上级部门
4. **编辑部门**：修改名称或调整上级
5. **删除部门**：仅当无子部门、无用户、无账号时可删

#### 4.5.4 角色权限说明

查看 5 级角色的权限范围说明，供管理员分配角色时参考。

#### 4.5.5 操作审计

查看平台操作日志，包括登录、用户/部门/账号变更等，支持分页浏览。

---

### 4.6 各角色典型操作流程

#### 系统管理员（admin）

```
登录 → 后台管理 → 维护院系部门 → 创建分级用户
     → 系统账号库 → 录入全校共享账号 → 设置可见范围
     → 操作审计 → 定期检查
```

#### 院系管理员

```
登录 → 后台管理 → 管理本院系用户与下级部门
     → 系统账号库 → 维护部门内共享账号
```

#### 科室负责人（it_manager）

```
登录 → 系统账号库 → 新增记录 → 填写本科室系统账号
     → 仅在本列表查看本人添加的记录
     → 可编辑/删除自己的记录
```

#### 普通教职工 / 只读用户

```
登录 → 系统账号库 → 查看被授权或部门共享的账号
     → 详情页查看密码（如有权限）
```

## 五、常见问题（FAQ）

**Q1：访问 5000 端口空白页？**  
A：macOS AirPlay 占用 5000，请使用 **5001** 端口。

**Q2：科室负责人为什么看不到别人的记录？**  
A：按设计，科室负责人仅可见**本人创建**的记录，这是权限隔离要求。

**Q3：忘记主密码怎么办？**  
A：联系系统管理员在后台重置；平台不存储可逆的用户密码。

**Q4：如何备份数据？**  
A：备份项目目录下的 `vault.db` 文件即可（SQLite 默认路径）。

**Q5：生产部署注意什么？**  
A：修改 `SECRET_KEY`、`ENCRYPTION_KEY`；启用 HTTPS；修改所有演示账号密码；建议使用 PostgreSQL。

## 六、环境变量（可选）

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `SECRET_KEY` | Flask 会话密钥 | 开发用固定值 |
| `ENCRYPTION_KEY` | 账号密码加密密钥 | 同 SECRET_KEY |
| `DATABASE_URL` | 数据库连接 | `sqlite:///vault.db` |

示例：

```bash
export SECRET_KEY="请替换为随机长字符串"
export ENCRYPTION_KEY="请替换为另一随机长字符串"
python run.py
```
